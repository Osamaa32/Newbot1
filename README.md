## Overview

**Complete Telegram multi-account management system** with a control bot, web dashboard, and powerful automation engine. Control everything from a single Telegram bot or through the web dashboard.

### Features

- **Control Bot** — Full control via Telegram bot with password authentication
- **Account Management** — Add, start, stop, remove accounts interactively
- **Auto-Reply System** — Automatic replies based on keywords
- **Message Forwarding** — Forward messages to target groups
- **Keyword Management** — Dynamic keyword configuration
- **Group Management** — Manage groups to join and monitor
- **Blocked Users** — Blacklist management
- **Filters** — Advanced message filtering system
- **Web Dashboard** — Real-time React dashboard with WebSocket updates
- **REST API** — Full API for external integrations
- **Health Monitoring** — Health checks and system logs
- **Backup System** — Export and import configuration

## Quick Start

### 1. Prerequisites

- Python 3.11+
- PostgreSQL 14+
- Telegram Bot Token (from [@BotFather](https://t.me/BotFather))

### 2. Local Development

```bash
# Clone and setup
git clone <repo-url>
cd telegram-control-system

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Setup environment
cp .env.example .env
# Edit .env with your values

# Run database migrations
python -c "
import asyncio
from shared.models import init_engine, create_tables, seed_defaults, SessionLocal
init_engine('postgresql://user:pass@localhost/telegram_bot_db')
asyncio.run(create_tables())
async with SessionLocal() as db:
    await seed_defaults(db)
"

# Start the system
python main.py
```

### 3. Docker Deployment

```bash
# Using Docker Compose
docker-compose up -d

# Or using Docker directly
docker build -t telegram-control .
docker run -d \
  -e BOT_TOKEN=your_token \
  -e ADMIN_PASSWORD=your_password \
  -e DATABASE_URL=postgresql://... \
  -p 8000:8000 \
  -p 8080:8080 \
  telegram-control
```

### 4. Railway Deployment

1. Fork this repository
2. Create new project on [Railway](https://railway.app)
3. Add PostgreSQL database
4. Set environment variables:
   - `BOT_TOKEN` — From @BotFather
   - `ADMIN_PASSWORD` — Your admin password
   - `DATABASE_URL` — Auto-generated by Railway
5. Deploy!

## Environment Variables

| Variable | Description | Required |
|----------|-------------|----------|
| `BOT_TOKEN` | Telegram bot token from @BotFather | Yes |
| `ADMIN_PASSWORD` | Password for admin authentication | Yes |
| `OWNER_ID` | Your Telegram user ID | No |
| `DATABASE_URL` | PostgreSQL connection string | Yes |
| `REDIS_URL` | Redis connection (optional) | No |
| `API_PORT` | API server port (default: 8000) | No |
| `HEALTH_PORT` | Health check port (default: 8080) | No |
| `ENVIRONMENT` | development/staging/production | No |
| `LOG_LEVEL` | DEBUG/INFO/WARNING/ERROR | No |

## Bot Commands

### Authentication
- `/start` — Start authentication

### Account Management
- `/accounts` — View all accounts with controls
- `/addaccount` — Add new account (interactive wizard)
- `/startacc <phone>` — Start account
- `/stopacc <phone>` — Stop account
- `/removeacc <phone>` — Remove account
- `/restartacc <phone>` — Restart account
- `/setmode <phone> <mode>` — Change mode (forward/reply/both)

### Statistics
- `/stats` — System statistics
- `/status` — Engine status
- `/health` — Health check

### Keywords
- `/keywords` — Manage keywords
- `/addkw <word>` — Add keyword
- `/delkw <word>` — Delete keyword
- `/listkw` — List keywords

### Groups
- `/groups` — Manage groups
- `/addgroup <link>` — Add group
- `/listgroups` — List groups

### Settings
- `/settings` — View settings
- `/set <key> <value>` — Update setting
- `/replyset <text>` — Set auto-reply
- `/replyshow` — Show auto-reply

### Security
- `/block <user_id>` — Block user
- `/unblock <user_id>` — Unblock user
- `/blocked` — List blocked users

### System
- `/refresh` — Refresh configuration
- `/restart` — Restart all accounts
- `/backup` — Create backup
- `/help` — Show help

## API Endpoints

### Accounts
- `GET /api/accounts` — List accounts
- `POST /api/accounts` — Create account
- `POST /api/accounts/{phone}/start` — Start account
- `POST /api/accounts/{phone}/stop` — Stop account
- `DELETE /api/accounts/{phone}` — Delete account

### Groups
- `GET /api/groups` — List groups
- `POST /api/groups` — Add group
- `DELETE /api/groups/{id}` — Delete group

### Keywords
- `GET /api/keywords` — List keywords
- `POST /api/keywords` — Add keyword
- `DELETE /api/keywords/{id}` — Delete keyword

### Settings
- `GET /api/settings` — List settings
- `PUT /api/settings` — Update setting

### Logs
- `GET /api/logs` — System logs
- `GET /api/reply-logs` — Reply logs

### WebSocket
- `WS /ws` — Real-time updates
