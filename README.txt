 3. خطوات التثبيت
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

الخطوة 1: إنشاء قاعدة بيانات PostgreSQL
─────────────────────────────────────────
  CREATE DATABASE botdb;
  CREATE USER botuser WITH PASSWORD 'your_password';
  GRANT ALL PRIVILEGES ON DATABASE botdb TO botuser;

الخطوة 2: إنشاء البوت من BotFather
────────────────────────────────────
  1. افتح @BotFather في تلجرام
  2. أرسل /newbot
  3. اتبع التعليمات للحصول على BOT_TOKEN
  4. احفظ التوكن

الخطوة 3: إعداد ملف البيئة
────────────────────────────
  1. انسخ ملف .env.example إلى .env:
     cp .env.example .env

  2. عدل الملف:
     BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrSTUvwxyz
     ADMIN_PASSWORD=password123
     DATABASE_URL=postgresql://botuser:password@localhost:5432/botdb

الخطوة 4: التثبيت
─────────────────
  pip install -r requirements.txt

الخطوة 5: التشغيل
─────────────────
  python bot_ultimate_v6.py

  ▪ ستظهر رسالة: "Bot is running!"
  ▪ اذهب للبوت في تلجرام وارسل /start

 📱 1. إدارة الحسابات
═══════════════════════════════════════════════════════════════

هذا القسم للتحكم الكامل في الحسابات العاملة.

── عرض الحسابات ─────────────────────────────────────────────
  ▪ يعرض كل الحسابات مع:
    - رقم الهاتف
    - الحالة (🟢 نشط / ⏸️ متوقف / 🔴 خطأ / ⏳ معلق)
    - الوضع (forward / reply / both)
    - آخر اتصال
  ▪ أزرار لكل حساب:
    ▶️  تشغيل الحساب
    ⏸️  إيقاف الحساب
    🗑  حذف الحساب نهائياً
    📊  عرض حالة مفصلة
    🔗  انضمام للجروبات

── إضافة حساب جديد ──────────────────────────────────────────
  ▪ معالج تفاعلي خطوة بخطوة:
    1. أرسل رقم الهاتف (مع كود الدولة): +966500000000
    2. أرسل API ID (رقم من my.telegram.org)
    3. أرسل API Hash (نص من my.telegram.org)
    4. أرسل Target Group ID (رقم المجموعة)
    5. اختر الوضع: forward / reply / both
    6. أدخل كود OTP الذي يصل للحساب
    7. إذا كان الحساب محمي بـ 2FA، أدخل كلمة المرور
  ▪ بعد النجاح، الحساب يبدأ يعمل مباشرة!

── تشغيل/إيقاف الكل ─────────────────────────────────────────
  ▪ ▶️ تشغيل الكل: تشغيل كل الحسابات المتوقفة
  ▪ ⏸️ إيقاف الكل: إيقاف كل الحسابات النشطة

── إيقاف الانضمام ───────────────────────────────────────────
  ▪ ⏹ إيقاف أي عملية انضمام جارية فوراً

── حالة الجروبات ────────────────────────────────────────────
  ▪ يعرض لكل حساب: في كم جروب هو عضو
  ▪ وكم جروب خارجها

── انضمام لجروبات ──────────────────────────────────────────
  ▪ اختر حساب → ينضم تلقائياً لكل الجروبات المخزنة
  ▪ يمكن إيقافه في أي وقت


═══════════════════════════════════════════════════════════════
 💬 2. إدارة الردود
═══════════════════════════════════════════════════════════════

الردود هي الرسائل التي يرسلها البوت تلقائياً.

── الردود المباشرة (Direct Reply Triggers) ──────────────────
  ▪ الجمل التي إذا وردت في رسالة، البوت يرد تلقائياً
  ▪ مثال: "ابي مساعده" → البوت يرد
  ▪ يمكن إضافة/حذف/عرض

── الردود المحظورة (Blocked Phrases) ────────────────────────
  ▪ الجمل التي إذا وردت، البوت لا يرد حتى لو فيها كلمات مفتاحية
  ▪ تستخدم لمنع الرد على رسائل معينة

── الردود التلقائية (Auto-Reply Messages) ───────────────────
  ▪ رسائل متناوبة يرسلها البوت
  ▪ ينتقل بينها دورياً (round-robin)
  ▪ مثال: رسالة ترحيب متغيرة

── الرد الافتراضي (Default Reply) ────────────────────────────
  ▪ الرسالة الأساسية التي ترسل للمستخدم
  ▪ يمكن تخصيصها بمتغيرات:
    {first_name}  → اسم المستخدم
    {last_name}   → اسم العائلة
    {username}    → المعرف (@username)
    {user_id}     → الرقم التعريفي
  ▪ مثال: "مرحباً {first_name}، كيف أقدر أساعدك؟"


═══════════════════════════════════════════════════════════════
 🔑 3. الكلمات المفتاحية
═══════════════════════════════════════════════════════════════

الكلمات التي يبحث عنها البوت في الرسائل الواردة.

── العرض ─────────────────────────────────────────────────────
  ▪ يعرض كل الكلمات المفتاحية (30 أولى + العدد الكلي)

── الإضافة ────────────────────────────────────────────────────
  ▪ أرسل كلمة أو أكثر (كل سطر كلمة)
  ▪ البوت يراقب هذه الكلمات في كل الرسائل

── الحذف ─────────────────────────────────────────────────────
  ▪ أرسل الكلمة أو الكلمات للحذف

── أمثلة على الكلمات الافتراضية ─────────────────────────────
  "ابي مساعده" / "واجب" / "مشروع" / "بحث" / "بروجكت"
  "case study" / "بوربوينت" / "عذر طبي" ...


═══════════════════════════════════════════════════════════════
 🛡 4. الفلاتر والإعدادات
═══════════════════════════════════════════════════════════════

فلاتر التجاهل — الرسائل التي تطابق هذه الفلاتر يتم تجاهلها.

── الفلاتر المتاحة ───────────────────────────────────────────
  ▪ mention    — تجاهل الرسائل التي تحتوي على @إشارة
  ▪ links      — تجاهل الرسائل التي تحتوي على روابط
  ▪ digits     — تجاهل الرسائل التي تحتوي على أرقام
  ▪ private    — تجاهل المحادثات الخاصة
  ▪ outgoing   — تجاهل الرسائل الصادرة (مني)
  ▪ bots       — تجاهل رسائل البوتات
  ▪ admins     — تجاهل رسائل المشرفين والمنشئين
  ▪ word_count — تجاهل الرسائل التي تتجاوز عدد كلمات معين

── التحكم ─────────────────────────────────────────────────────
  ▪ اضغط على أي فلتر → يتبدل بين ✅ مفعل و ❌ معطل
  ▪ فوري ومباشر!

  ▪ أرسل user_id للإلغاء

── بحث ────────────────────────────────────────────────────────

═══════════════════════════════════════════════════════════════
 ⚙️ 9. الإعدادات المتقدمة
═══════════════════════════════════════════════════════════════

── عرض الإعدادات ─────────────────────────────────────────────
  ▪ عرض كل الإعدادات الحالية مع القيم والوصف

── تعديل إعداد ───────────────────────────────────────────────
  ▪ أرسل: key value
  ▪ مثال: rate_limit_max 6

── إعادة تعيين ────────────────────────────────────────────────
  ▪ إعادة إعداد للقيمة الافتراضية
  ▪ أرسل اسم الإعداد

── أهم الإعدادات ─────────────────────────────────────────────
  ▪ rate_limit_max        — حد الردود لكل مستخدم (افتراضي: 4)
  ▪ rate_limit_window     — فترة الحد بالثواني (افتراضي: 3600)
  ▪ cb_failure_threshold  — حد أخطاء الانضمام (افتراضي: 5)
  ▪ cb_recovery_timeout   — فترة الانتظار بعد الخطأ (افتراضي: 1800)
  ▪ join_delay_base       — تأخير الانضمام الأساسي (افتراضي: 30)
  ▪ join_delay_random     — تأخير إضافي عشوائي (افتراضي: 30)
  ▪ fallback_group_id     — مجموعة الاحتياط (افتراضي: -1002353780992)
  ▪ word_count_limit      — حد عدد الكلمات (افتراضي: 17)
  ▪ fuzzy_threshold       — حد البحث الفازي (افتراضي: 80)

 6. النصائح والحيل
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 للحصول على API ID و API Hash:
   1. اذهب إلى https://my.telegram.org
   2. سجل دخول برقم هاتفك
   3. اذهب إلى API development tools
   4. أنشئ تطبيق جديد
   5. احفظ api_id و api_hash

💡 لمعرفة Group ID:
   1. أضف @userinfobot للمجموعة
   2. أو استخدم بوت مثل @RawDataBot
   3. يبدأ Group ID بـ -100

💡 الوضع (Mode):
   ▪ forward — يوجه الرسائل فقط
   ▪ reply   — يرد على المستخدمين فقط
   ▪ both    — يوجه ويرد (الافتراضي)

💡 الحد من الردود:
   ▪ إذا مستخدم تجاوز rate_limit_max ردود في rate_limit_window
   ▪ يتم حظره تلقائياً من الردود

💡 Circuit Breaker:
   ▪ إذا فشل حساب 5 مرات متتالية في الانضمام
   ▪ يتوقف تلقائياً لمدة 30 دقيقة
   ▪ لحماية الحساب من الحظر

❌ "BOT_TOKEN not set"
   → تأكد من ملف .env وإنه يحتوي على BOT_TOKEN

❌ "DATABASE_URL not set"
   → تأكد من إعداد قاعدة البيانات وملف .env

❌ "كلمة المرور خاطئة"
   → تأكد من ADMIN_PASSWORD في ملف .env

❌ "Session غير صالح"
   → الحساب يحتاج إعادة إضافة، استخدم "إضافة حساب جديد"

❌ "FloodWait"
   → تلجرام يطلب انتظار. البوت يتعامل معها تلقائياً

❌ البوت لا يستجيب
   → تأكد من تشغيل البرنامج
   → تحقق من السجلات في ملف bot.log
